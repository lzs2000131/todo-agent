## 🚀 快速启动

### 方法1: 开发模式(推荐用于测试)

```bash
# 启动开发服务器
npm run tauri dev
```

这将:
- 启动Vite开发服务器
- 打开Tauri应用窗口
- 支持热重载

### 方法2: 生产构建

```bash
# 构建生产版本
npm run tauri build
```

构建产物位于:
- **macOS DMG**: `src-tauri/target/release/bundle/dmg/`
- **macOS App**: `src-tauri/target/release/bundle/macos/`

## 🔧 配置应用

### 必需配置

1. **OpenAI API Key** (截图识别功能)
   - 启动应用后,点击设置图标
   - 输入你的 OpenAI API Key
   - 保存设置

### 可选配置

2. **AWS S3 云端同步** (可选)
   - 在设置中配置:
     - S3 Bucket 名称
     - AWS Region
     - Access Key ID
     - Secret Access Key
   - 开启"启用云端同步"开关

## 🎯 核心功能

### 1. 截图识别 (Cmd+Shift+T)

**使用步骤:**
1. 按下 `Cmd+Shift+T` (或点击相机图标)
2. 选择要截取的屏幕区域
3. AI 自动识别待办事项
4. 预览并选择要添加的任务
5. 点击"添加待办"

**提示:**
- 截图内容应包含清晰的任务列表
- AI 可识别标题、描述、优先级、标签、截止日期
- 支持多个待办事项同时识别

### 2. 待办管理

**基础操作:**
- **添加**: 点击"添加新待办..."
- **编辑**: 点击待办项的编辑图标
- **删除**: 点击删除图标
- **完成**: 点击圆圈标记

**分类管理:**
- 点击侧边栏"添加分类"
- 选择图标和颜色
- 为待办分配分类

### 3. 智能提醒

**自动提醒时机:**
- 截止前 24 小时
- 截止前 1 小时
- 到期时刻
- 自定义提醒时间

**注意:**
- 需要允许系统通知权限
- 应用会每30分钟自动检查

### 4. 云端同步

**如何工作:**
- 配置 S3 后自动启用
- 每60分钟自动备份
- 应用启动时立即同步
- 智能冲突解决(保留最新)

## 🐛 故障排查

### 应用无法启动

```bash
# 清理端口
lsof -ti:1420 | xargs kill -9

# 重新启动
npm run tauri dev
```

### 截图功能不工作

- 确保已配置 OpenAI API Key
- 检查系统权限(屏幕录制权限)
- 查看控制台错误信息

### 通知不显示

- 检查系统通知权限
- 确保应用在前台或后台运行
- macOS 系统偏好设置 > 通知

### 云端同步失败

- 验证 S3 配置是否正确
- 检查网络连接
- 确认 AWS 凭证有效

## 💡 使用技巧

### 最佳实践

1. **截图识别**
   - 截取包含明确任务的屏幕
   - 避免截取过于复杂的图像
   - 文字清晰的截图效果最好

2. **分类组织**
   - 使用不同颜色区分工作/生活
   - 善用标签进行多维度组织
   - 定期清理已完成任务

3. **优先级管理**
   - 高优先级用于紧急重要事项
   - 中优先级用于常规任务
   - 低优先级用于待办清单

4. **云端同步**
   - 定期检查同步状态
   - 重要数据建议手动备份
   - 多设备使用相同 S3 配置

## ❓ 常见问题

**Q: 是否支持 Windows/Linux?**
A: 当前主要支持 macOS,但 Tauri 理论上支持跨平台,可以修改配置后构建。

**Q: 截图识别需要网络吗?**
A: 是的,需要调用 OpenAI API,需要网络连接。

**Q: 数据存储在哪里?**
A: 本地使用 SQLite 数据库,云端可选择存储到 S3。

**Q: 是否收费?**
A: 应用本身免费,但需要:
- OpenAI API 使用费用(按调用量计费)
- AWS S3 存储费用(可选,按使用量计费)
